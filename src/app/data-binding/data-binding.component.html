Data Binding in Angular

One-Way Data Binding

One-way data binding in Angular refers to the flow of data from the component to the view or from the view to the component. This can be done using:

  • Interpolation
  • Property Binding
  • Attribute Binding
  • Event Binding (View to Component)
  • Style Binding
  • Class Binding

Interpolation

<h1>{{ title }}</h1>

In the above example, the value of the component's title property is displayed in the view.

Property Binding

<img [src]="imageUrl" alt="Image description">

Property binding is used to bind the value of a component property to an element property in the view.

Event Binding

<button (click)="handleClick()">Click me</button>

Event binding allows you to listen to events such as clicks and key presses. The method handleClick() is called when the button is clicked.

Two-Way Data Binding

Two-way data binding in Angular is a combination of property binding and event binding. It allows you to synchronize data between the component and the view.

Example with ngModel

<input [(ngModel)]="name" placeholder="Enter your name">
<p>Hello, {{ name }}!</p>

In this example, the value of the name property in the component is bound to the input field and the paragraph element. Any changes to the input field are reflected in the name property and vice versa.
